# TD 装备掉落系统设计

**日期：** 2026-06-07

## 目标

为当前 Warcraft III WurstScript TD 项目设计一套高可玩性的塔装备系统。该系统需要满足以下目标：

- 装备主要服务于 `塔` 的成长，而不是英雄或共享全队面板
- 装备掉落、拾取、穿戴、拆卸、合成的规则尽量直观，优先复用 Warcraft III 原生物品理解
- 系统要有明确的 `赌性`，但不能复杂到压过 TD 主循环
- 装备要能强化现有五个塔系的分工与流派，而不是取代塔本身定位
- 首版要能在当前 `4 路、16 波、5 塔系、5 级完美升级` 的骨架上稳定落地

## 当前项目上下文

当前项目已经具备多人 TD 主循环骨架：

- `4` 名防守玩家
- `4` 条路线，合流到共享中路终点
- `16` 波同步推进
- `5` 个塔系
- 每个塔系最高 `5` 级
- 已存在 `5` 级完美升级机制
- 塔与怪的物编由 compiletime 生成

当前相关模块包括：

- `TDConfig`
- `TDState`
- `TDWaves`
- `TDSpawn`
- `TDGame`
- `TDTowers`

因此，本设计只处理装备系统本身，不重做现有游戏骨架。

## 范围

### 本次设计包含

- 塔装备的掉落、拾取、转移、穿戴、拆卸
- 装备品质与随机升级规则
- 白绿蓝紫四档装备体系
- 普通怪、精英怪、Boss 的掉落节奏
- 与五个塔系的适配方向
- 首发装备池建议
- 平衡护栏与首轮测试重点

### 本次设计不包含

- 套装效果
- 洗练、重铸、锁定
- 多前后缀随机词缀系统
- 装备仓库分页
- 自动拾取
- 英雄装备系统
- 无尽模式专属装备

## 核心设计原则

- 规则简单，决策有深度
- 优先复用 Warcraft III 原生“地上物品、单位物品栏、单位间转移物品”的理解
- 装备系统服务于 `养一座主力塔`、`临场换装`、`废件赌博升阶`
- 白装负责构筑底座，绿蓝紫负责制造方向感和高潮
- 允许玩家之间互相丢装，但不做掉落归属保护
- 首版尽量避免 UI 菜单化和按钮膨胀

## 核心玩法循环

单局中的装备主循环如下：

1. 敌人死亡后有概率掉落装备到地面
2. 工人像捡普通物品一样把装备捡进自己的物品栏
3. 玩家决定：
   - 直接把装备给某座塔穿上
   - 暂时留在工人物品栏里，等待合成
   - 扔到地上给队友或别的塔线重新分配
4. 主力塔逐步凑满 `3` 个装备槽，形成清晰流派
5. 废件或过渡件进入工人物品栏前几格，通过合成按钮进行赌博升阶
6. 波次压力变化时，玩家可把塔上的装备拆下，重新调配给别的塔

这套循环的核心乐趣来自：

- 掉什么
- 给谁穿
- 是稳穿还是拿去赌
- 波间要不要拆装重组

## 装备持有与穿戴模型

### 工人

- 工人拥有原生物品栏
- 装备掉在地上后，工人可直接拾取
- 工人可以像原生单位转移物品一样，把装备给目标塔
- 工人只保留 `1` 个额外主动按钮：`合成`

### 塔

- 每座塔拥有 `3` 个装备格
- 塔不能主动拾取地面装备
- 塔只能通过 `工人给予` 的方式接收装备
- 塔满 `3` 格时不能接收新装备，必须先拆卸
- 塔可执行两种拆卸行为：
  - `拆到地上`
  - `拆给附近工人`

### 玩家之间的流通

- 怪物掉落到地上的装备 `立即公共可捡`
- 玩家捡起后若再丢到地上，其他玩家也可捡
- 不做掉落归属保护期

## 装备槽与叠加规则

- 每座塔最多 `3` 件装备
- 任意装备可放入任意装备格
- 允许同名装备重复穿戴
- 同一座塔的第 `3` 件同名装备效果衰减为 `50%`
- 装备带来的总冷却缩减上限为 `40%`
- 装备带来的总额外射程上限建议为 `+500`

设计意图：

- 允许“三攻速”“双金币一幸运”这类明确构筑
- 避免完全禁止重复穿戴带来的限制感
- 同时防止少数词条通过三重叠加直接失控

## 合成系统

### 合成入口

- 工人拥有 `1` 个合成按钮
- 合成时只检查工人物品栏前 `3` 格

### 合成规则

- 如果前 `3` 格中有 `3` 件同品质装备，则执行 `3 合 1`
- 如果前 `3` 格中没有 `3` 件同品质，但存在 `2` 件同品质装备，则执行 `2 合 1`
- 如果品质不一致，或数量不满足要求，则不合成

执行优先级：

- 优先检查是否满足 `3 合 1`
- 若不满足，再检查是否存在可用的 `2 合 1`
- 若前 `3` 格为“`2` 件同品质 + `1` 件不同品质”，则只消耗那 `2` 件同品质材料

### 合成结果

- `2 合 1`：`65%` 成功，生成 `下一品质随机装备`
- `3 合 1`：`100%` 成功，生成 `下一品质随机装备`
- 材料只要求 `同品质`，不要求同名
- 紫装为终点，不能继续升阶
- 合成失败时材料全部消失
- 合成成功的产物直接返回工人物品栏

### 随机结果策略

首版建议采用 `全池随机`，不做材料方向加权。

原因：

- 规则更纯粹，赌性更直接
- 首版实现更简单
- 更容易观察玩家是否喜欢纯随机升阶

如果后续实测挫败感过高，再考虑加入轻量方向加权。

## 品质结构

首版使用四档品质：

- `白色`：基础单维度装备
- `绿色`：更强的双属性或强化单属性装备
- `蓝色`：能明显定义一座塔流派的装备
- `紫色`：局内高潮装备，稀有但不过度破坏平衡

建议整体价值大致按以下档位增长：

- 白：`1.0`
- 绿：`1.7`
- 蓝：`2.8`
- 紫：`4.2`

该比例用于整体强度校准，不要求每个词条都严格按固定倍率成长。

## 掉落节奏设计

### 每路每波掉落预算

为避免地上装备过多，建议每一路每一波设置掉落预算上限：

- `1-3` 波：每路每波最多 `1` 件
- `4-7` 波：每路每波最多 `2` 件
- `8-11` 波：每路每波最多 `3` 件
- `12-15` 波：每路每波最多 `4` 件
- `16` 波：每路至少 `1` 件高品质奖励，并叠加 Boss 掉落

### 普通怪掉率

建议普通怪掉率按波次平滑提升：

- `1-3` 波：`10%-14%`
- `4-7` 波：`15%-20%`
- `8-11` 波：`20%-24%`
- `12-15` 波：`24%-28%`
- `16` 波护卫怪：`25%-30%`

普通怪主要掉落：

- 前中期以 `白装` 为主
- `7` 波后允许极低概率直出 `绿装`
- 普通怪不直出 `紫装`

### 精英波节点

结合当前现有波表，重点节点如下：

- `8` 波精英：
  - 必掉 `1` 件绿装
  - 额外概率掉 `1` 件白装
- `12` 波双精英：
  - 每只精英高概率掉绿装
  - 整波额外进行 `1` 次蓝装判定
- `15` 波决战精英：
  - 至少掉 `1` 件蓝装
  - 并补 `1-2` 件白/绿装，给玩家最后一轮冲品质

### Boss 波奖励

`16` 波 Boss 建议：

- 必掉 `1` 件蓝装
- `50%` 概率额外掉 `1` 件蓝装
- `12%-15%` 概率直出 `1` 件紫装

### 品质主要来源

- 白装：普通怪为主
- 绿装：精英掉落 + 白装合成
- 蓝装：中后期精英/Boss 掉落 + 绿装合成
- 紫装：蓝装合成为主，Boss 极低概率直掉

## 五个塔系的装备方向

### 弓箭手

- 偏好：攻击、射程、攻速、处决
- 角色：稳定主 C
- 典型构筑：`攻击 + 攻速 + 射程`

### 兽人战士

- 偏好：攻击、重击、破甲、Boss 增伤
- 角色：精英/Boss 单点爆破
- 典型构筑：`攻击 + 破甲 + 终结`

### 人族步兵

- 偏好：冷却、控制强化、射程
- 角色：控场与辅助核心
- 典型构筑：`冷却 + 控制强化 + 射程`

### 小鹿

- 偏好：攻速、冷却、毒伤、减速强化
- 角色：持续压制与团队放大器
- 典型构筑：`攻速 + 毒伤 + 功能强化`

### 食尸鬼

- 偏好：攻速、处决、击杀金币、掉宝增益
- 角色：收割与发育
- 典型构筑：`经济 + 收尾 + 攻速`

## 白装词条建议范围

首版数值建议如下：

- `短弓`：攻击 `+8 / +14 / +22 / +34`
- `鹰眼镜`：射程 `+90 / +150 / +230 / +340`
- `迅击手套`：攻速 `+12% / +20% / +32% / +48%`
- `冷凝符文`：冷却缩减 `8% / 12% / 18% / 26%`
- `猎金指环`：击杀额外金币 `+1 / +2 / +3 / +4`
- `破甲尖刺`：命中减甲 `1 / 2 / 3 / 4`
- `处决匕首`：半血以下增伤 `12% / 20% / 30% / 45%`
- `毒雾囊`：持续伤害 `6 / 10 / 16 / 24` 每秒
- `稳固图腾`：异常强度 `+10% / +16% / +25% / +40%`

`拾荒袋` 作为掉装相关词条，不建议首版给出过高收益，避免发育流成为唯一最优解。

## 与现有完美升级系统的关系

装备系统不应压过现有 `5 级完美升级`，而应与之形成叠加高潮：

- 先把塔养到 `5` 级
- 再给成型塔喂高品质装备
- 完美升级塔优先成为高品质装备的最佳承载者

目标体验：

- 完美升级提供“塔本体仪式感”
- 装备提供“构筑与赌博爽点”
- 两者相加形成“这一路的主核塔”记忆点

## 平衡护栏

首版建议强制保留以下约束：

- 冷却缩减总上限 `40%`
- 额外射程总上限 `+500`
- 击杀额外金币对 Boss 不生效
- 掉宝相关效果不影响 Boss 保底奖励
- 控制增强只强化已有控制，不把无控塔改造成控制塔
- 第 `3` 件同名装备效果衰减为 `50%`
- 紫装不可继续合成

## 玩家交互与表现

### 交互原则

- 不新增多余按钮
- 尽量复用原生捡物、给物、丢物的体验
- 只额外提供 `1` 个合成按钮

### 合成按钮反馈

建议在合成结果时提供清晰提示：

- `合成成功：白装 x3 -> 绿色 佣兵钱袋`
- `合成失败：白装 x2 已损毁`

### 掉落与品质表现

- 白装：普通掉落特效
- 绿/蓝/紫：更明显的颜色文字和地面特效
- 精英/Boss 掉高品质装备时给出醒目提示

## 首发版本建议

第一版建议只做以下闭环：

- 地面掉落
- 工人捡装备
- 工人给塔穿装备
- 塔 `3` 槽
- 塔拆到地上/拆给工人
- `2 合 1 65%`
- `3 合 1 100%`
- 白绿蓝紫四档
- 普通怪/精英/Boss 分层掉落

首版先不做：

- 套装
- 重铸
- 洗练
- 锁装备
- 复杂随机词缀
- 自动拾取
- 专属职业限制

## 测试重点

### 首轮体验验证

- 玩家是否能自然理解“捡起、给塔、拆下、合成”
- 白装是否足够好懂，不会看起来都一样
- `2 合 1` 是否有赌感但不至于只剩挫败
- 是否能自然形成主力塔流派
- 玩家之间是否会出现主动丢装、换装、喂主核的行为

### 首轮平衡验证

- 经济装是否过强
- 控制装是否让步兵失控
- 攻速 + 毒伤是否让小鹿/食尸鬼滚雪球过快
- `2 合 1` 和 `3 合 1` 是否都具备合理吸引力
- Boss 直掉高品质装备是否足够有高潮感

## 模块拆分建议

建议后续实现时新增并拆分为以下模块：

- `TDItems`
  - 装备定义、品质、名字、说明、颜色、词条
- `TDDrops`
  - 掉落判定、掉落预算、普通/精英/Boss 掉表
- `TDInventory`
  - 工人物品栏与塔三槽的穿脱、转移、拆卸逻辑
- `TDCrafting`
  - `2 合 1`、`3 合 1`、品质升阶与随机产物

现有模块的职责建议保持：

- `TDGame` / `TDSpawn`
  - 怪物死亡后调用掉落接口
- `TDTowers`
  - 读取塔当前装备并应用到属性和技能逻辑
- `TDState`
  - 保存装备相关运行时状态

## 后续扩展方向

待首版验证通过后，第二版优先考虑：

- 事件型装备
- 更鲜明的辅助型蓝紫装
- Boss 专杀件
- 风险更高的经济件
- 轻量方向加权合成

## 成功标准

如果首版能达到以下状态，就说明系统已经成立：

- 玩家愿意在波间拆装重配
- 玩家会讨论“这件给谁”
- 玩家会在 `2 合 1` 和 `3 合 1` 之间做真实取舍
- 玩家能记住几件核心装备名字
- 一局结束后，玩家能复盘“这局我养的是哪座塔，为什么成型”

这说明装备系统已经不只是加属性，而是真正改变了 TD 的局内决策结构。
